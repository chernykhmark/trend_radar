# Профиль пользователя

## Роль
Системный аналитик и инженер данных. Совмещает проектирование систем с прикладной работой с данными и запуском продуктов.

## Профессиональные интересы
- **Проектирование систем**: архитектура, интеграции, событийные модели, моделирование данных, ETL/ELT, хранилища, потоковая обработка.
- **Прикладная работа с данными**: SQL/Python, аналитика, DWH, dbt, Airflow, метрики, качество данных.
- **Запуск MVP и продуктовые идеи**: быстрая проверка гипотез, no-code/low-code, продуктовая аналитика, unit-экономика.
- **Автоматизация**: скрипты, пайплайны, AI-агенты, LLM-интеграции в рабочие процессы.
- **Профессиональный путь**: карьерные траектории, переходы между ролями (BA → SA → DE → PM), софт-скиллы для технарей.

## Целевые сегменты бизнеса
SMB, стартапы, продуктовые команды. Контекст — небольшие команды, ограниченные ресурсы, акцент на скорость и ROI.

## Что НЕ интересно
- Хардкорный low-level (драйверы, ядро ОС, реверс).
- Геймдев, графика, embedded.
- Корпоративный enterprise без прикладной ценности (бюрократия, ITIL ради ITIL).
- Чисто академические теоретические статьи без применимости.
- Криптовалюты как спекуляция.

## Критерии релевантности
Высокая оценка — если пост даёт **применимое знание** в перечисленных интересах ИЛИ описывает **боль/кейс/тренд** в целевых сегментах. Низкая — если узкоспециально вне профиля или поверхностно.
